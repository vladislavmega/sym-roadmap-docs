Review of current icinga web implementation

Current icingaweb written on deprecated technologies, on frontend it's spaghetti-like jquery code.
On backend zend1 php framework (not supported by vendor since 2016-09-28). New php developers don't learn zend, old php developers don't want write on zend 1 in 2018.

Study Vue.js as core frontend framework

I select Vue.js as core frontend (from vue, react, angular).
Reasons: vue uses html-like templates (react uses own jsx syntax and should be compiled before).
Vue can be putted on page and start works (angular requires TypeScript from microsoft, and should be compiled too).
With vue you can rewrite old project part-by-part.
All core stuff (vuex, vue-router, vue-cli, vue-ssr). written and supports by vue core team (react team supports only react. redux, router and other important libs are third-party).
Vue works very fast (same speed as react, sometimes faster).

Study Vuex (vue state controller)

Next logic step, just manage application state in one place.

Study GraphQL (with optional pubsub)

Great solution instead rest api:
Supports pub/sub.
Queries and mutations protected and typized by schema. (all unnecessary field ignored by schema).
Every string in schema can be commented and then you can generate api documentation from comments.
Has clients for most popular languages and platforms (JavaScript, php, android, ios, etc..).

Check selected stack on dhcp and statusmap

As we know it's looks very nice and works really fast. Code clean and solid to understand and extend.

Implement modular platform with users and roles

Current implementation supports:
Build and start only backend server.
Build and start only frontend server (for example case where we have one centralized backend and many frontends).
Build and start both.
Use http or https protocol.
Send push notifications.
Enable/disable modules (Dhcp, statusmap, inventory, users).

Rewrite part-by-part icingaweb

With current our platform implementation we can rewrite icingaweb part-by-part, and replace rewrited parts in icingaweb to our versions.

Remove php icingaweb, use our, "Symbotic icingaweb"

When all parts be rewrited we can just remove old icingaweb.
